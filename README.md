# GoA2 v2

这是 Guards of Atlantis II 电子版的新项目骨架。目标是先把多人联机、地图、队长出生点、回合/轮结构、移动交互做稳，再逐步填完整卡牌规则。

## 启动

```powershell
conda activate C:\Users\29383\Documents\StudyFile\personal_project\GoA2\goa2_v2\.conda
python app.py 3207
```

浏览器打开：

```text
http://localhost:3207
```

## 当前实现

- 4 个浏览器标签页测试，每个标签页使用独立 `sessionStorage` token。
- 房间创建、入座、设队长。
- 开局由双方队长点击本队英雄出生点，为本队英雄依次分配出生点。
- 开始游戏前必须 4 人入座且 4 名英雄都有出生点。
- 暗选、翻牌、按先攻结算。
- 当前测试牌：每名英雄 5 张牌，其中 3 张主要动作是击杀相邻敌方小兵，1 张主要动作是攻击相邻敌方英雄，1 张主要动作是攻击任意敌方单位；每张牌也可以作为次要移动，或放弃效果。
- 英雄被攻击时进入防御响应：日志显示小兵修正后的最终伤害，被攻击者可弃置防御值不低于伤害的手牌，否则死亡。死亡者轮到自己按先攻结算时，先自己选择复活点，再执行牌。
- 击杀奖励：近战/远程小兵 2 金，重型小兵 4 金，英雄按被击杀者等级给金币。
- 破平币：同先攻且跨队时，当前破平币面对应队伍先结算，然后翻面。
- 地图来自 `C:\Users\29383\Documents\StudyFile\personal_project\GoA2\goa2_v2\data\map.json`。服务启动时只读取一次；修改后需要重启服务器。启动时会保留中部主连通块，并把地形统一设为 `region: "terrain"`。
- 移动：可点击移动按钮后高亮合法格，再点击格子执行移动。
- 快速移动：当前区域和目标区域都无敌方英雄/小兵，且两个区域相同或相邻。
- 产兵：战线在中部时，双方所有“中部区域里的小兵出生点”同时生成对应小兵；战线推进后改用对应方向的下一片区域。
- 轮末兵线骨架：比较双方小兵数量，少的一方额外移除；一方小兵全灭时推进战线并重生小兵。
- 轮末自动升级英雄等级：按当前英雄等级作为费用，金币够就连续升级。
- 普通移动按绕墙路径计算步数；卡牌预留 `canPhaseThroughWalls` 字段给未来穿墙效果。
- 双方生命点各 7，复活消耗生命点的服务端结构已存在。

## 已知未完成

- 攻击/防御/弃牌交互目前是测试版，只实现相邻英雄攻击、防御弃牌、死亡。
- 还没有队长手动选择“具体哪位英雄出生/复活”的 UI；目前按本队待出生顺序分配。
- 还没有队长手动选择小兵移除、出生点被阻挡时选择相邻格的 UI；当前为自动处理。
- 卡牌具体数值和完整牌效尚未逐张实现；当前使用测试牌。
- 升级阶段已迁入 v2：轮末先结算兵线，再自动按英雄等级花金币升级，随后玩家为每次升级选择红/绿/蓝卡的两个方向之一。三色均 2 级后才能升 3，三色均 3 后获得紫色大招占位。英雄等级上限为 8。

## 规则摘要

- 一轮包含 4 个回合。每回合玩家暗选 1 张手牌，全部选择后翻牌结算。
- 若玩家因弃牌等原因无牌可打，应跳过该回合。一轮结束后回收弃牌。
- 英雄升级花费等于升级前英雄等级。三色技能各升到 2 级总花费为 1+2+3=6。
- 三色均到 2 才能升 3；三色均到 3 后获得大招。
- 重型小兵必须在其它友方小兵都死亡后才能被移除或击杀。
- 战线推进：一方小兵全部移除后，所有小兵清空，战线向先被清空的一方推进。推进方填战线标记，先填 3 个或推进到出生点获胜。
